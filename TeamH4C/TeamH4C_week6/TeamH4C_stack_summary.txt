 C 언어에서 변수를 할당하는 방법에는 지역변수로 할당하는 방법, 전역변수로
할당하는 방법이 있다. 여기서 지역변수는 함수가 실행될 때마다 생성되는 영역에
위치하고 함수의 실행이 종료되면 더이상 유효하지 않은 변수를 말한다. 그리고
전역변수는 한 번 생성하면 어떤 함수에서든 참조할 수 있는 변수이다. Stack은
C 언어에서 지역변수의 할당을 구현하기 위해 사용된다.
 Stack은 선입후출 (Last In First Out, LIFO)의 구조를 가지는 자료구조의
한 형태이다. Stack에서는 push 연산으로 데이터를 추가하고, pop 연산으로
데이터를 인출한다. 그리고 이 데이터를 추가하고, 인출할 위치를 지정하기 위해
stack pointer가 사용된다.
 함수 호출이 발생하면 stack은 컴파일 시 계산된 크기만큼 할당되고 할당된
만큼 확장된다. 따라서 함수 호출이 발생함에 따라 stack은 자라게 되는 것이다.
그리고 이 과정은 함수 프롤로그에서 수행된다. 함수 프롤로그는 다음과 같은
명령어로 구성된다.

-------------------------------------------------------
push rbp       ;; 이전 stack의 base pointer를 저장
mov rbp, rsp   ;; rsp 레지스터 값을 rbp에 대입하여 새로운 stack의
               ;; base pointer 생성
sub rsp, ...   ;; rsp에서 컴파일 시 계산된 크기만큼 뺄셈을 수행하여
               ;; 새로운 stack 프레임을 할당
-------------------------------------------------------

함수의 호출이 발생하였을 때 stack이 할당된다면 함수 실행의 종료가 발생하였을
때는 stack을 다시 복구해야 한다. 이 과정은 함수 에필로그에서 수행되고,
다음과 같은 명령어로 구성된다.

-------------------------------------------------------
leave   ;; mov rsp, rbp; pop rbp와 같은 기능을 수행
        ;; mov rsp, rbp는 rbp 레지스터의 값을 rsp에 대입하여 할당된
	;; stack을 다시 할당되기 이전 상태로 복원
	;; pop rbp는 저장되어 있던 stack base pointer를 인출하여
	;; rbp 레지스터에 저장함으로써 stack을 복구
ret     ;; stack에 있는 값을 pop하여 instruction pointer에 저장
        ;; instruction pointer가 이 주소의 명령어를 실행함에
	;; 따라 호출자의 코드가 실행됨
-------------------------------------------------------

 Stack base pointer는 할당된 stack에 위치한 지역변수의 주소를 계산하기
위한 base pointer이고 앞서 살펴보았듯이 stack을 복구하는데 사용된다.
Stack base pointer는 stack의 base pointer이므로 이 stack에 위치한
지역변수는 base pointer에 적절한 인덱스를 더하는 방법으로 참조될 수 있다.
