---[ 2. 유니코드
 [2, p. 1]은 유니코드 표준(The Unicode Standard)이 쓰여진 문자와 텍스트를
위한 인코딩 표준이고, 유니코드 문자는 UTF-32 (32비트 형식),
UTF-16 (16비트 형식), UTF-8 (아스키 기반 시스템에서의 쉬운 사용을 위한
8비트 형식) 으로 나뉠 수 있다고 설명한다.

 유니코드 표준의 범위는 [2, p. 3]이 설명하며 그 범위는 주요 국제, 국가 표준과
산업계에서 사용하는 주요 문자 표준을 포함한다.

---[ 3. C에서의 유니코드 사용법
 [3, Sec. 7.26.12]는 extended multiplebyte와 wide character
utilities에 대해서 설명하지만 그것이 어떤 값을 저장할지는 명시하지 않는다.
그러나 [4, Sec. 7.27]은 유니코드를 사용하기 위한 type과 functions를
기술하고 있다. 하지만 현재에도 많은 implementation들이 C11을 지원하지 않고
C99를 지원하는 경우가 있기 때문에 그것들이 제공하는, 유니코드와 관련된 기능을
확인해볼 필요가 있다.

 상기 서술한 C에서의 문제들은 C 프로그램이 유니코드 문자를 대상으로 연산을
수행하는 경우에 적용될 수 있다. 그러나 [1]이 제시한 문제와 같이 출력만 하는
경우에는 그 대상이 유니코드인지 아닌지는 C 표준이 아닌 OS와 터미널 환경이
확인해야할 문제가 된다. 그래서 OS와 터미널 환경이 지원하는 문자 집합에 따라
다른 문자가 출력될 수 있다.

---[ 4. 문제 풀이
 [3, Sec. 6.4.3]은 universal character names이 기본 문자 집합에 포함되지
않는 문자를 표기할 때 사용될 수 있음을 설명한다. 따라서 (기본 문자 집합에
포함되지 않는) 유니코드 문자를 터미널 환경이 출력하도록 만들기 위해 universal
character names를 사용할 수 있다. 이로부터 다음과 같은 코드를 작성할 수 있다.

-------------------------------------------------------
#include <stdio.h>

int main()
{
    printf("\u250C\u252C\u2510\n");
    printf("\u251C\u253C\u2524\n");
    printf("\u2514\u2534\u2518");
    return 0;
}
-------------------------------------------------------
