---[ 1. 1010번
 [1]은 다음과 같이 문제를 제시한다.

=======================================================
재원이는 한 도시의 시장이 되었다. 이 도시에는 도시를 동쪽과 서쪽으로
나누는 큰 일직선 모양의 강이 흐르고 있다. 하지만 재원이는 다리가 없어서
시민들이 강을 건너는데 큰 불편을 겪고 있음을 알고 다리를 짓기로 결심하였다.
강 주변에서 다리를 짓기에 적합한 곳을 사이트라고 한다. 재원이는 강 주변을
면밀히 조사해 본 결과 강의 서쪽에는 N개의 사이트가 있고 동쪽에는 M개의
사이트가 있다는 것을 알았다. (N ≤ M)

재원이는 서쪽의 사이트와 동쪽의 사이트를 다리로 연결하려고 한다. (이때 한
사이트에는 최대 한 개의 다리만 연결될 수 있다.) 재원이는 다리를 최대한
많이 지으려고 하기 때문에 서쪽의 사이트 개수만큼 (N개) 다리를 지으려고
한다. 다리끼리는 서로 겹쳐질 수 없다고 할 때 다리를 지을 수 있는 경우의
수를 구하는 프로그램을 작성하라.

입력
입력의 첫 줄에는 테스트 케이스의 개수 T가 주어진다. 그 다음 줄부터 각각의
테스트케이스에 대해 강의 서쪽과 동쪽에 있는 사이트의 개수 정수
N, M (0 < N ≤ M < 30)이 주어진다.

출력
각 테스트 케이스에 대해 주어진 조건하에 다리를 지을 수 있는 경우의 수를
출력한다.
=======================================================

---[ 2. 조합론
 조합은 n개의 원소를 갖는 집합에서 k개의 원소를 갖는 부분집합의 개수를
찾는 방법이다. 이러한 조합은 파스칼의 삼각형으로부터 다음과 같은
특성을 가짐을 알 수 있다.

=======================================================
nCr = (n-1)C(r-1) + (n-1)Cr (단, r <= n)
=======================================================

---[ 3. 문제 풀이
 [1]이 제시하는 문제에서 구하는 것은 동쪽 사이트 m개 중에서 n개를 선택하여
서쪽의 사이트를 잇는 다리의 개수를 구하는 것과 같다. 따라서 이는 m개 중에서
n개를 선택하는 조합 문제이다. 여기서 주어진 n, m의 조합을 계산하기 위해
미리 계산된 테이블을 만들 수 있다. 이때 0 < n <= m < 30이라는
제한조건으로부터 30 x 30인 조합테이블을 만들면 됨을 알 수 있다. 그리고
조합테이블을 계산할 때는 2.에서 제시한 특성을 활용하면 될 것이다. 이들로부터
다음과 같은 코드를 작성할 수 있다.

=======================================================
#include <stdio.h>
#include <stdint.h>

int main()
{
    int t, n, m, i, j;
    uint64_t ctab[31][31];
    
    ctab[0][0] = ctab[0][1] = -1; /* 0C0, 0C1은 쓰지 않는다 */
    ctab[1][0] = ctab[1][1] = 1;
    for (i = 2; i != 31; i++) {
        ctab[i][0] = 1;
        for (j = 1; j != 31; j++)
            ctab[i][j] = ctab[i - 1][j - 1] + ctab[i - 1][j];
    }
    scanf("%d", &t);
    while (t != 0) {
        scanf("%d %d", &n, &m);
        printf("%ld\n", ctab[m][n]); /* mCn */
        t--;
    }
    return 0;
}
=======================================================
